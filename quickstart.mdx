---
title: "Get up and running with Cash Reconciliation Workflow"
description: "Clone the repo, seed five demo cases, and run the full AI-assisted cash reconciliation workflow locally in under five minutes — no API key required."
---

This quickstart walks you through cloning the repository, copying the environment file, running the database migration and seed, and opening the live UI. The app ships with a deterministic `RuleBasedProvider` that produces realistic AI suggestions without any network calls or API credentials, so you can explore the full compliance workflow immediately.

<Note>
  The `RuleBasedProvider` requires no API key. It applies the variance tolerance and escalation criteria from `CaseType.rulesConfig` deterministically, with no network calls and no charge. You can run the entire demo — including AI suggestions — without touching `OPENAI_API_KEY`.
</Note>

<Steps>
  <Step title="Clone and install dependencies">
    Clone the repository and install dependencies for all three workspaces (`shared`, `backend`, `frontend`) from the repo root:

    <CodeGroup>

    ```bash npm install
    git clone https://github.com/MihirM9/cash-reconciliation-workflow.git
    cd cash-reconciliation-workflow
    npm install
    ```

    ```bash npm workspaces install
    git clone https://github.com/MihirM9/cash-reconciliation-workflow.git
    cd cash-reconciliation-workflow
    npm install --workspaces
    ```

    </CodeGroup>
  </Step>

  <Step title="Configure the environment">
    Copy the example environment file into place. The defaults work out of the box — no changes are required to run the demo:

    ```bash
    cp backend/.env.example backend/.env
    ```

    The key variables in `backend/.env` are:

    | Variable | Default | Purpose |
    |---|---|---|
    | `DATABASE_URL` | `file:./dev.db` | SQLite database path |
    | `PORT` | `4000` | Backend API port |
    | `OPENAI_API_KEY` | _(empty)_ | Leave blank to use `RuleBasedProvider` |
    | `AI_MODEL` | `gpt-4o-mini` | OpenAI model, only used when a key is set |
    | `FRONTEND_ORIGIN` | `http://localhost:5173` | CORS allowed origin for the frontend |

    <Tip>
      To switch to a real LLM, set `OPENAI_API_KEY` in `backend/.env` and restart the server. The backend logs which provider it chose on startup, and every AI suggestion permanently records `modelName` and `runId` in the audit trail — so you always know which model produced a given output.
    </Tip>
  </Step>

  <Step title="Migrate the database and seed demo data">
    Run the Prisma migration to create the SQLite schema, then seed the five demo cases:

    ```bash
    npm run prisma:migrate --workspace backend
    npm run seed
    ```

    The seed creates four users (Ava Analyst, Omar Ops, Casey CCO, Alex Admin) and five cases at different workflow stages, covering every audit event type. See the [demo walkthrough](/demo-walkthrough) for a full description of each case.
  </Step>

  <Step title="Run both apps">
    Start the backend API and frontend in parallel from the repo root:

    ```bash
    npm run dev
    ```

    The backend starts on `http://localhost:4000` and the frontend on `http://localhost:5173`. Both reload automatically on file changes.
  </Step>

  <Step title="Pick a user in the Acting as dropdown">
    Open **http://localhost:5173** in your browser. The first thing to do is pick a user from the **Acting as** dropdown in the top-right corner.

    This controls the `x-user-id` header the frontend sends on every request, and therefore which actions you can perform. Start as **Ava Analyst** to generate an AI suggestion, then switch to **Omar Ops** to submit a reviewer decision.
  </Step>
</Steps>
